[SQL] 실무에서 자주 쓰는 SQL TOP 5

실무에서 자주 쓰는 SQL TOP 5 실제 서비스에서는 단순 조회보다 다양한 조건과 패턴이 함께 사용된다. 자주 사용되는 SQL 패턴을 정리한다. 1. 최신 데이터 조회 SELECT * FROM posts ORDER BY created_at DESC LIMIT 10; 가장 최근 데이터 10개 조회 게시판, 로그 조회 등에서 사용 2. 특정 기간 데이터 조회 SELECT * FROM orders WHERE created_at >= '2026-01-01' AND created_at < '2026-02-01'; 기간별 데이터 조회 통계, 리포트에서 자주 사용 3. 중복 제거 SELECT DISTINCT user_id FROM orders; 중복된 값을 제거하고 고유 값만 조회 4. 그룹별 집계 SELECT user_id, COUNT(*) AS order_count FROM orders GROUP BY user_id; 사용자별 주문 횟수 계산 5. 조건에 따른 그룹 필터링 SELECT user_id, COUNT(*) AS order_count FROM orders GROUP BY user_id HAVING COUNT(*) >= 3; 주문이 3건 이상인 사용자 조회 정리 ORDER BY + LIMIT → 최신 데이터 조회 WHERE → 조건 필터링 DISTINCT → 중복 제거 GROUP BY → 집계 HAVING → 그룹 조건 필터링

[SQL] HAVING - 그룹 조건 필터링

HAVING HAVING은 GROUP BY로 묶인 결과에 조건을 적용할 때 사용한다. 1. 필요성 GROUP BY 이후 조건을 적용해야 하는 경우가 있다. 2. 기본 구조 SELECT 컬럼, 집계함수 FROM 테이블명 GROUP BY 컬럼 HAVING 조건; 3. 예제 SELECT department, COUNT(*) FROM employees GROUP BY department HAVING COUNT(*) >= 2; 인원이 2명 이상인 부서만 조회한다 4. WHERE vs HAVING -- WHERE SELECT * FROM employees WHERE salary > 3000; -- HAVING SELECT department, COUNT(*) FROM employees GROUP BY department HAVING COUNT(*) >= 2; WHERE: 그룹화 이전 조건 HAVING: 그룹화 이후 조건 정리 HAVING은 GROUP BY 결과에 조건을 적용한다 집계 함수와 함께 사용한다 WHERE와 역할이 다르다
